                    CHANGELOG — PROYECTO GPS (Gestor Documental)

Resumen: durante esta semana el proyecto incorporó la subida masiva de
documentos, la eliminación de documentos y carpetas, autenticación con
Google, previsualización y descarga de documentos, un pipeline completo de
CI/CD (GitHub Actions + Jenkins) con suite de tests automatizados, y la
corrección de múltiples errores detectados en pruebas de estrés y revisión
de código. Se integró además el dataset "data2" al gestor documental.

1. FUNCIONES AÑADIDAS
--------------------------------------------------------------------------------

[05-07-2026 | commit 276bff1] Subida masiva y eliminación
  - Subida masiva de documentos: permite cargar múltiples archivos a la vez
    desde el modal de subida (arrastrar y soltar incluido).
  - Eliminación de documentos y carpetas desde la interfaz, con popover de

[05-07-2026 | commit 414a603] Mejoras tras pruebas de estrés
  - Optimización de la carga de etiquetas por documento: nuevo endpoint
    por lotes (POST /api/user/documents/my-tags) que consulta las etiquetas
    de muchos documentos en una sola petición.
  - Ampliación de la analítica organizacional (resúmenes y tendencias).

[06-07-2026 | commit 9f610ae] Testing automatizado e integración continua
  - Suite de tests del backend con Jest + Supertest (autenticación,
    documentos, salud del servicio) que se ejecuta sin necesidad de
    MongoDB gracias al modo fallback en memoria.
  - Pipeline de CI en GitHub Actions (.github/workflows/ci.yml): tests del
    backend, lint y build del frontend, y validación de imágenes Docker en
    cada push y pull request.
  - Integración de Jenkins para testing continuo: Jenkinsfile declarativo
    y entorno Jenkins reproducible con Docker Compose (carpeta jenkins/),
    siguiendo las prácticas DevOps vistas en clase (CI/CD e infraestructura
    como código).

[06-07-2026 | commit 9f610ae] Inicio de sesión con Google
  - Botón oficial "Continuar con Google" en la página de login (Google
    Identity Services).
  - Nuevo endpoint POST /api/auth/google que verifica criptográficamente
    el token de Google y crea la cuenta automáticamente en el primer
    acceso.

[06-07-2026 | commit 9f610ae] Integración del dataset data2
  - Script de importación (npm run seed:data2) que carga el dataset de
    issues de GitHub (8.703 archivos de 16 repositorios de computación
    cuántica) como documentos cifrados, organizados por repositorio.
    Importación idempotente y por lotes.

[06-07-2026 | pendiente de commit] Previsualización y descarga de documentos
  - Vista previa de documentos en un modal según el tipo de archivo:
    imágenes, PDF, texto/código, audio y vídeo, con descarga directa para
    los formatos no previsualizables. Nuevo endpoint seguro
    GET /api/user/documents/:id/preview (sirve el archivo descifrado
    "inline" respetando los permisos de acceso).
  - Botones explícitos de "Vista previa" y "Descargar" en el menú de
    acciones de cada documento (antes la descarga solo existía por doble
    clic). La descarga conserva ahora la extensión original del archivo,
    por lo que funciona con cualquier tipo de documento subido.

[06-07-2026 | commit 9f610ae] Documentación y configuración
  - README principal del repositorio con estructura, comandos y guía de
    CI/CD; guía de credenciales (CREDENCIALES.md); configuración de

2. BUGS CORREGIDOS
--------------------------------------------------------------------------------

[05-07-2026 | commit 414a603] Correcciones de pruebas de estrés
  - Arreglos de rendimiento y estabilidad en Home y Shared al trabajar con
    grandes volúmenes de documentos (carga de etiquetas por lotes en lugar
    de una petición por documento).

[06-07-2026 | commit 9f610ae] Correcciones de backend
  - Registro de usuarios sin MongoDB: los usuarios creados en modo
    fallback quedaban sin identificador interno (_id), lo que invalidaba
    su sesión y bloqueaba todas las rutas protegidas (error 401).
  - Caída del proceso ante errores de base de datos: las rutas de registro
    e inicio de sesión no capturaban excepciones; un fallo de conexión
    podía tumbar el servidor completo.
  - Error 500 con identificadores malformados: las rutas de documentos,
    carpetas, conjuntos, etiquetas y organizaciones lanzaban una excepción
    interna ante un ID inválido; ahora responden 404 correctamente.
  - Crash en la consulta de cuenta (/api/user/account/:id) cuando el
    usuario no existía.
  - Cabecera de descarga insegura: el saneamiento del nombre de archivo
    era inefectivo (permitía inyección de comillas y saltos de línea en
    Content-Disposition).
  - El log de conexión imprimía la URI de MongoDB con usuario y contraseña
    en texto plano.
  - Conexión a MongoDB Atlas: el error querySrv ECONNREFUSED se debía a
    que el DNS de la red rechaza consultas SRV; se migró la cadena de
    conexión al formato estándar con hosts explícitos del clúster.

[06-07-2026 | commit 9f610ae] Correcciones de frontend y build
  - Dependencia npm espuria "nginx" eliminada del package.json (paquete
    ajeno al proyecto que se instalaba sin uso alguno).
  - Configuraciones duplicadas de plantilla eliminadas (vite.config.ts,
    eslint.config.js y App.tsx): el lint tomaba la configuración
    equivocada y App.tsx referenciaba un módulo inexistente.
  - 289 errores de formato (ESLint/Prettier) corregidos en todo el código
    del frontend.
  - El workflow de despliegue a Azure fallaba en cada push desde abril
    (25 ejecuciones fallidas) por apuntar a un directorio inexistente
    ("Proyect-Gest-Docum"); se corrigió la ruta y se pasó a ejecución
    manual.
  - Los lockfiles (package-lock.json) estaban excluidos de git, lo que
    rompía "npm ci" en Docker y en cualquier entorno de CI.

[06-07-2026 | pendiente de commit] Corrección de interfaz
  - En una de las vistas de documentos, el menú de acciones no recibía la
    función de eliminar, por lo que el botón "Eliminar" fallaba al hacer
    clic.
